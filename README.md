## What I am building now

PatriciAI and OpenClaw are becoming a practical operating layer for agents, memory, tools, documentation, speech, workflows, and human-guided automation. The work is intentionally local-first where privacy and control matter, API-first where integrations need to scale, and governed by staging discipline before anything reaches production.

Current build areas:

- **Agent workspaces and skills** - reusable instructions, tool patterns, and guardrails so assistants can do real work without becoming unbounded.
- **Memory and documentation** - daily notes, curated long-term memory, Obsidian-backed project records, retrieval, and context that survives beyond one chat.
- **Local AI services** - OpenAI-compatible local inference, embeddings, reranking, speech-to-text, and text-to-speech running through controlled service boundaries.
- **Tool orchestration** - GitHub, Discord, n8n, files, schedulers, browser automation, media tools, and project-specific APIs wired into useful workflows.
- **Omi and mobile integration** - Android tooling, emulator validation, wearable/mobile assistant paths, and the early bridge between personal capture and PatriciAI operations.
- **Infrastructure maturity** - Proxmox/Linux lab work, dashboards, service routing, credential stewardship, backups, and production-readiness habits that make the interesting parts dependable.
- **Governed automation** - external actions, secrets, public posts, destructive work, and private context handled with permissioning, status clarity, and recoverable execution.

The goal is not autonomy for its own sake. The goal is capability with oversight.

## Operating beliefs

- Most modernization problems are **execution problems first**.
- Automation without control is just **faster chaos**.
- Blind trust is not an AI strategy.
- Technology should never be a stumbling block between staff and the mission.
- Useful systems explain themselves before touching production.
